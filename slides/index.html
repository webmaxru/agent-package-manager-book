Turn your team's patterns
into a package

A package is just a folder: an apm.yml + your .apm/ skills, prompts & rules. Publish it as a trusted source for everyone.

Govern at fleet scale · the payoff

Org intent,
enforced at the door.

Policy flows org → team, tighten-only. Legal bumps a rule once; every repo picks it up on next install — and CI won't merge without it.

terminal · the unbypassable gate 
$ apm audit --ci
✓ 9 checks passed · policy: atlas
✗ unpinned dep → exit 1  (PR blocked)
atlas/.github · org baseline 
# tighten-only: org → team, never looser
enforcement: block
dependencies:
  require_pinned_constraint: true
# blocks untrusted transitive
# MCP servers at install time
⏱ 2:30 · total 26:00

This is the answer to the cold open

  • Remember "org intent can't reach the point of use"? Here it does. Policy is discovered from the org, merged tighten-only, enforced at install AND re-checked in CI.
  • "A developer can pass --force locally. CI cannot. The org's baseline is the one thing nobody can quietly skip."
  • Rollout is humane: start in warn mode, watch the audit pass-rate climb, then flip to block. An untrusted transitive MCP server is a sub-supplier you never vetted — policy blocks it too.

Leader aside

  • The metric isn't "how many manifests" — it's audit pass-rate and drift trend across the fleet.

Start your supply chain

Three steps to a trusted supply chain

  • Publish one trusted package.  Lift your best rules & prompts into a folder with an apm.yml.
  • Pin it in every repo.  apm install restores it on Copilot, Claude, Cursor…
  • Gate it in CI.  apm audit --ci verifies provenance and blocks violations.
