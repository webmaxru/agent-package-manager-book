> package declares no `dependencies:`, so its lockfile records only

## The producer manifest (keys apm v0.23.1 accepts)
`name`, `version` are **required**; `description`, `author`, `license`,
`repository`, `keywords`, `targets`, `includes` are read and surfaced.
**`type: hybrid` is a real enum but RESERVED/INERT** at v0.23.1 — behaviour is
inferred from package content and it is not carried into `plugin.json`. Unknown
top-level keys are silently preserved (no error). Full key-acceptance table and
the diff vs the running example:
[../../../content/research/10-becoming-a-producer-reference.md](../../../content/research/10-becoming-a-producer-reference.md).

## Reproduce (offline)
```powershell
# PATH refresh (Windows), then work in a scratch dir OUTSIDE the repo:
$env:Path = [System.Environment]::GetEnvironmentVariable("Path","Machine") + ";" + [System.Environment]::GetEnvironmentVariable("Path","User")
$root = "$env:TEMP\apm-ch10-repro"; Remove-Item -Recurse -Force $root -ErrorAction SilentlyContinue
Copy-Item -Recurse "<repo>\backend\examples\ch10\meridian-standards" $root
Set-Location $root

apm install                     # auto-discovers .apm/ via `includes: auto`; compiles to 3 targets; exit 0
apm pack --dry-run --verbose    # -> "Would write … .github\plugin\plugin.json" + .claude-plugin; NO bundle
apm pack --archive -o dist -v   # -> writes the two plugin.json in-tree; dist/ stays EMPTY (plugin.json mode)
```

## `apm pack` routing (why this package produces plugin.json, not a bundle)
`apm pack` decides output from which blocks `apm.yml` declares:

| `apm.yml` has… | `apm pack` writes |
|---|---|
| `dependencies:` | a **bundle** — `build/<pkg>/` (+ `--archive` `.zip`) |
| `marketplace:` | `.claude-plugin/marketplace.json` |
| `targets:` incl. `claude`/`copilot` | **plugin.json** in-tree (`.github/plugin/`, `.claude-plugin/`) |
| none of the above | **"Nothing to pack"** (exit 1) |

`meridian-standards` has `targets:` but **no `dependencies:`**, so it lands in
plugin.json mode. To ship it, use a **git ref**
(`apm install meridian-finance/meridian-standards#v1.0.0`) or add a
`dependencies:` closure to produce a `--archive` bundle. Bundle mode **rejects
local-path dependencies** and needs **remote** refs → `SKIPPED-needs-network`.

## Notes / SKIPPED-needs-network
- **Offline round-trip (proven):** `apm install "<local meridian-standards>"`
  into a fresh repo integrates the `.apm/` primitives and caches the package
  under `apm_modules/_local/`. The recorded dep is an absolute path — use a
  relative `./` path or a git ref for a portable manifest.
- **`apm publish`** requires `apm experimental enable registries` (disabled by
  default) and uploads via `PUT /v1/packages/{owner}/{repo}/versions/{version}`
  → `SKIPPED-needs-network`.
- **`apm marketplace init`** scaffolds a `marketplace:` block offline; resolving
  `packages[].source` against remotes → `SKIPPED-needs-network`.
- **`--target` on `apm pack` is deprecated**; **`--format apm` is legacy** and
  `apm install` rejects it — use the default `--format plugin`.
